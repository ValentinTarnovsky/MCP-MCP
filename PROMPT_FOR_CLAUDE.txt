## Overview
Build an MCP server that helps Minecraft Java plugin developers check the latest documentation and versions of common dependencies.

## Core Features

### 1. Dependency Documentation Lookup Tool
Create a tool called `get_dependency_docs` that can fetch and return the latest documentation URLs and version info for these dependencies:
- **LuckPerms** - https://luckperms.net/wiki & https://github.com/LuckPerms/LuckPerms
- **Paper API** - https://docs.papermc.io/paper & https://repo.papermc.io/repository/maven-public/
- **Bukkit/Spigot** - https://hub.spigotmc.org/javadocs/bukkit/

- **Item-NBT-API** - https://github.com/tr7zw/Item-NBT-API/wiki

- **CoreProtect API** - https://github.com/PlayPro/CoreProtect/
- **mc-MenuAPI** - https://github.com/MonGen-s-Cave/mc-MenuAPI

### 2. Project Dependency Scanner Tool
Create a tool called `scan_project_dependencies` that:
- Accepts a project path as input
- Scans `build.gradle`, `build.gradle.kts`, and `pom.xml` files
- Extracts all dependencies with their current versions
- Returns a structured list of dependencies found

### 3. Version Checker Tool
Create a tool called `check_latest_versions` that:
- For each known dependency, fetches the latest version from Maven Central, JitPack, or Paper's maven repo
- Compares against the project's current versions (if provided)
- Returns upgrade recommendations

### 4. Full Project Analyzer Tool
Create a tool called `analyze_plugin_project` that:
- Scans "C:\Users\tarno\Desktop\OkiMC-Plugins" folder (or a custom path)
- Finds all gradle/pom files in subprojects
- Extracts all unique dependencies across projects
- Returns a comprehensive dependency report

## Technical Requirements
- Use TypeScript with the MCP SDK (@modelcontextprotocol/sdk)
- Include proper error handling
- Support both Gradle (Groovy & Kotlin DSL) and Maven pom.xml parsing
- Cache dependency info to avoid repeated lookups
- Use fetch/axios for HTTP requests to check latest versions
